# 招待キャンペーン v2

招待キャンペーンは、開催ごとに成績を分けるシーズン制です。既存の通常招待・Bump/Up・財布・宿・マーケットは置き換えません。

## 状態

- **待機中**: 新しいシーズンを開始できます。
- **開催中**: 新規招待を記録します。
- **精算中**: 新規受付を止め、既存対象者の24時間定着とVC達成だけを精算します。

受付終了後の精算期間は、24時間定着条件と同じ時間です。期限を迎えると10分ごとのバックグラウンド処理で自動確定し、結果を履歴へ保存します。

## 報酬成立のタイミング

Discordサーバーへ入っただけではキャンペーン参加報酬を支給しません。招待された人が参加ボタンを押し、経済圏への初回参加が完了した時点で成立します。

VC条件は、その人がキャンペーン対象になった時点の累計VC時間を基準値として保存し、以後に増えた時間だけを数えます。

## データの扱い

- シーズン開始時に参加記録・ランキングを新しくします。
- 前シーズンの集計と対象者記録は履歴へ保存します。
- Campaign Ticket残高はシーズンをまたいで保持します。
- 旧式の全消去ボタンは廃止しています。
- 交換商品は未整備のため、Campaign Ticket交換は停止中です。閲覧と残高保持だけを行います。

## 管理操作

`/管理` → `Campaign管理` から操作します。

1. `新シーズン開始`で確認画面を開く。
2. 開催中は`参加者確認`で参加・定着・VC進捗を見る。
3. `受付を終了`で精算期間へ移行する。
4. 通常は精算期限まで待つ。緊急時のみ`今すぐ確定`を使う。
5. `履歴`で確定済みシーズンの結果を見る。

## 検証

```bash
npm run check
npm run smoke
```

`src/campaign-v2-test.js` は、参加ボタン前の支給防止、VC基準値、二重支給防止、精算期間、シーズン履歴、Ticket引継ぎ、破壊的リセット廃止を確認します。
