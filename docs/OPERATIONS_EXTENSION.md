# 運営拡張: ニックネーム整理とログインボーナス

この拡張は `src/discord-entry-operations.js` から読み込まれ、既存の `discord-entry-manual-join.js` より前に安全な追加処理を登録します。マーケット、宿、カジノなど既存の本体処理は置き換えません。

## ニックネーム一括削除

`/管理` の運営パネルに `ニックネーム一括削除` ボタンが追加されます。

1. ボタンを押すと、現在カスタムニックネームが付いている通常ユーザーとBotの件数を取得します。
2. `通常ユーザーのみ削除` または `Botを含めて削除` を選びます。
3. 実行結果として、削除件数、権限順位によるスキップ件数、失敗件数を表示します。

実行できるのは `ECONOMY_ADMIN_IDS` に含まれるユーザー、Administrator、またはサーバー管理権限を持つユーザーです。Bot側には `ニックネームの管理` 権限が必要です。

サーバー所有者、Botより上位または同順位のロールを持つメンバーなど、Discord上で変更できない対象は自動的にスキップされます。元のニックネームはバックアップしないため、確認画面を経由しない実行経路はありません。

`DISCORD_LOG_CHANNEL_ID` が設定されている場合、実行者と結果件数を監査ログとして送信します。同じサーバーでの同時実行はロックされます。

## ログインボーナス

ログインボーナスはローリング20時間制から、日本時間の日付基準へ変更されます。

- 毎日 0:00 JST に受取可能になります。
- 同じ日本時間の日付では1回だけ受け取れます。
- 前日に受け取っていれば連続日数が増えます。
- 1日以上空いた場合、連続日数は1日目へ戻ります。
- 既存の `lastDaily` と `streak` はそのまま移行判定に利用します。

報酬計算は既存仕様を維持します。

```text
基本 600 Ris
+ 連続日数 × 120 Ris（最大 1,800 Ris）
+ ランダム 0〜180 Ris
+ 常連カード所持時 250 Ris
```

ホームパネルには受取可否、現在の連続日数、次回受取時刻が表示されます。受取済みの日はホームのログボボタンが無効化されます。

## 検証

```bash
npm run check
npm run smoke
```

`src/operations-extension-test.js` では、日本時間の日付境界、同日二重受取、連続日数の加算とリセット、常連カード加算、ニックネーム対象抽出、運営パネルのDiscord 5ボタン制限補正を検証します。
